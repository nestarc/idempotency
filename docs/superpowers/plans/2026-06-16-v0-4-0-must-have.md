# v0.4.0 Must-Have Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the v0.4.0 must-have scope from `docs/superpowers/specs/2026-06-16-v0-4-0-scope-spec.md`.

**Architecture:** Keep the current decorator/module/interceptor/storage model. Add new public option types in `idempotency-options.interface.ts`, resolve route-level overrides in `IdempotencyInterceptor`, and reuse the existing storage contract by passing `processingTtl` to `create()` and replay `ttl` to `complete()`. Observability is emitted from interceptor state-machine boundaries and must never change request outcomes.

**Tech Stack:** NestJS 10/11, TypeScript 5.4, RxJS 7, Jest 29, Memory/Redis/Postgres storage adapters.
